# Media capture plan — RTS Commander Doctrine

Portfolio and store pages should show **discipline**, not feature spam. This plan targets a **~90 second** narrated demo once doctrine slices land; until then, capture what is **honestly implemented** and label WIP clips.

---

## 90-second demo structure (target)

| Segment | Time (approx.) | Content |
| --- | --- | --- |
| Hook | 0:00–0:08 | Title card: unofficial mod; one-line thesis (“formation is discipline”). |
| Problem | 0:08–0:18 | Vanilla battle: camera tied to agent; orders feel menu-forward. |
| Solution | 0:18–0:45 | RTS Commander Mode: read battlefield, issue native orders, markers. |
| Doctrine teaser | 0:45–1:12 | Commander nucleus + rally + spacing (when implemented). |
| Close | 1:12–1:30 | Roadmap honesty + link to repository docs. |

---

## Capture checklist

- [ ] **Resolution:** 1080p minimum; stable frame pacing.
- [ ] **HUD:** show key prompts unobtrusively (config keys on screen overlay is optional).
- [ ] **Audio:** separate voice track from game audio for editing.
- [ ] **Build stamp:** show `docs/version-lock.md` game version in opening or description.
- [ ] **Disclaimer:** “Unofficial mod” in description text.

## Editing notes

- Prefer **one continuous take** per clip; cut on beats, not mid-input.
- Avoid music that reads as “trailer cliché” for portfolio submissions unless brand asks for it.
- Keep **WIP** labels in filenames (`WIP_rally_v0`).
