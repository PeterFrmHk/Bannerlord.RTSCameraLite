# Diagrams

## Naming

`system-name.png` (or `system-name.svg` if your toolchain supports vector in-repo)

Examples:

- `doctrine-pipeline.png` — MissionView → doctrine modules → router → executor.
- `cavalry-sequence-states.png` — state machine overview for interviews.

## Style

- **High contrast** for projection and small screens.
- Label **adapter boundaries** explicitly (camera, formations, native orders).
- Date optional here; clips/screenshots carry the date for freshness instead.
